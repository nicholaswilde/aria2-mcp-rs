# Product Guidelines - aria2-mcp-rs

## Prose Style: Technical & Direct
- **Objective**: Provide precise and accurate information.
- **Rules**:
    - Use clear and unambiguous technical terminology.
    - Avoid conversational filler and unnecessary jargon.
    - Prioritize clarity and accuracy in all documentation and messages.

## Branding: Aria2 Aligned
- **Objective**: Maintain consistency with the established `aria2` identity.
- **Rules**:
    - Follow `aria2`'s color schemes and visual style where applicable.
    - Use recognized `aria2` icons and branding elements to signify compatibility.
    - Emphasize the project's role as a direct bridge to `aria2`'s functionality.

## UX Principles: Efficiency-First
- **Objective**: Enable fast and productive download management.
- **Rules**:
    - Focus on low-latency interactions for both the MCP server and CLI.
    - Streamline common tasks to require minimal user input or configuration.
    - Optimize resource usage to ensure the server remains lightweight and responsive.

## Error Handling: Detailed & Technical
- **Objective**: Facilitate quick diagnosis and resolution of issues.
- **Rules**:
    - Provide comprehensive error messages with relevant technical context.
    - Include detailed logs and, where appropriate, stack traces for deep debugging.
    - Ensure errors are clearly and immediately communicated through all interfaces.
